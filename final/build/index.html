Cursor Changes

Triggers

Cursor changes are triggered when the user hovers over the interactive area or presses down on the button.

Rules

  • The cursor changes to a custom dot with an outline when hovered over the button.
  • The cursor outline expands on hover and contracts on press.

Feedback

  • Visual: The cursor dot becomes slightly larger on hover and contracts to indicate pressing.
  • Animation: Transitions are smooth with a 0.15s ease effect to enhance user experience.

Loops & Modes

No additional loops or modes are defined for cursor changes. The interaction resets after each hover or press event.

Button Interactions

Triggers

The button interaction is triggered on hover and mouse down events.

Rules

  • The button edge slightly lifts on hover to indicate interactivity.
  • On mouse down, the button depresses halfway to show it is being pressed.
  • The button returns to its original position on mouse up.

Feedback

  • Visual: The button color changes to a bright yellow (#ffe01b) on hover and a deeper yellow (#ffc300) on press.
  • Haptic: A slight vibration may occur on press if the device supports haptic feedback.
  • Screen Transition: On mouse up, the interaction completes, and the user is navigated to a new screen. A div element temporarily slides in from the right side as a transitional effect.

Loops & Modes

The temporary div animation slides in from the right when the new screen loads, providing additional visual feedback. This div remains visible momentarily before sliding back out, marking the completion of the transition.

Slider Transition

Triggers

The slider transition is triggered when the form is successfully submitted, initiating a slide-in animation that overlays the entire page.

Rules

  • The slider element slides in smoothly from the right side of the screen upon form submission.
  • A close button (Ã—) is positioned in the top-right corner of the slider, allowing users to manually dismiss the overlay.
  • The slider remains visible until the user decides to close it using the close button.

Feedback

  • Visual: The slider moves seamlessly with a 2s ease-out effect, providing a smooth transition.
  • Interactive: The close button changes color on hover to indicate interactivity and can be clicked to slide the overlay out.

Loops & Modes

The slider transition is a one-time effect per successful form submission, resetting once the slider is closed. This allows the transition to be repeated on subsequent valid submissions.

Error Message Animations

Triggers

The error message animation is triggered when a user submits a form with invalid or incomplete data.

Rules

  • The error message appears below the relevant form field.
  • The message includes a brief explanation of the error.
  • The form field border changes color to red (#ff4d4d) to indicate an error.

Feedback

  • Visual: The error message fades in with a slight slide-up animation over 0.3s.
  • Animation: The form field shakes slightly to draw attention.

Loops & Modes

The error message remains visible until the user corrects the input. The shake animation occurs on every wrong submission to signal the user.

Notification Pop-ups

Triggers

The notification pop-up is triggered when a significant event occurs, such as receiving a new message or completing a task.

Rules

  • The pop-up appears at the bottom-right corner of the screen.
  • It displays a concise message and an icon representing the notification type.
  • The pop-up auto-dismisses after 5 seconds unless interacted with.

Feedback

  • Visual: The pop-up slides in from the bottom with a smooth transition of 0.4s.
  • Animation: Hovering over the pop-up pauses auto-dismissal and highlights the background.
  • Auditory: A soft chime plays when the pop-up appears if sound is enabled.

Loops & Modes

The notification operates in passive mode, appearing only when triggered by an event. It does not loop or repeat unless a new event occurs. Users can manually dismiss it by clicking a close icon.

Triggers

The close button is triggered when the user clicks on it while the slider overlay is active.

Rules

  • The close button (Ã—) is prominently placed in the top-right corner of the slider overlay.
  • On hover, the close button changes color to provide visual feedback indicating it is interactive.
  • Clicking the close button initiates a slide-out animation, dismissing the slider overlay.

Feedback
